Everything a real booking backend needs

JWT authentication

Register and log in to receive a signed token with 24-hour expiry. Every booking route is protected.

Role-based access

Users manage their own bookings; admins see and manage everything, including hard deletes.

Overlap validation

Conflicting time slots are rejected with a clean 409 Conflict — double-booking is impossible by design.

Interactive docs

Full OpenAPI 3.0 spec rendered with Swagger UI. Authorize once and fire real requests from the browser.

Soft cancel history

Cancelled bookings keep their record with a cancelled status, so the schedule stays auditable.

One-command deploy

Docker-ready with zero-config SQLite storage. Clone, run, and the whole stack is up in under a minute.
